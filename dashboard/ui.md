## UI 设计方案

### 设计原则

沿用现有代码库的视觉语言（IBM Plex Sans、slate 色阶、teal 强调色），在此基础上建立系统化的设计规范，确保 7 个页面之间视觉一致。

---

### 一、设计系统规范

#### 色彩体系

```
背景层级
  Page bg        #f1f5f9   slate-100
  Surface        #ffffff   white
  Surface-alt    #f8fafc   slate-50
  Overlay        #f1f5f9   同 Page bg，用于 hover 状态

边框
  Default        #e2e8f0   slate-200   0.5px
  Emphasis       #cbd5e1   slate-300   1px，用于分隔线

文字
  Primary        #0f172a   slate-900
  Secondary      #475569   slate-600
  Muted          #94a3b8   slate-400
  Link           #0d9488   teal-600

  Teal-600       #0d9488   主操作、激活状态

### 二、整体布局框架

```
┌─────────────────────────────────────────────────────────────┐
│  顶栏（48px）：Logo + 实验状态 + 语言切换                        │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                   │
│  侧边栏  │              页面内容区                             │
│  200px   │              (flex: 1)                            │
│  固定    │                                                   │
│          │                                                   │
└──────────┴──────────────────────────────────────────────────┘
```

**顶栏（新增）**

高度 48px，白底，下边框 `#e2e8f0`。

左：`🔬 Opinion Dynamics` 14px/500，点击返回首页。

中：实验状态胶囊（仅在运行/完成后出现）：
- 运行中：`⚙ 运行中  Step 234/500` — teal 底
- 完成：`✓ 已完成  500步 · 89事件` — green 底
- 停止：`■ 已停止  Step 321/500` — slate 底

右：语言切换 `中文 / EN`（两个文字按钮，当前语言加下划线）。

**侧边栏（重构）**

宽 200px，白底，右边框，内容区垂直滚动。分三个区域：

```
┌────────────────────┐
│  导航区（flex:1）   │
│  7 个导航项         │
│──────────────────  │
│  配置进度区         │
│  进度条 + 文字      │
│──────────────────  │
│  版本信息           │
└────────────────────┘
```

**导航项解剖**（高度 36px）：

```
[8px] [状态点6px] [8px] [图标14px] [8px] [文字13px] [auto] [8px]
```

激活态：整行 `background: #f0fdfa`，左侧 2px teal 竖线，文字 primary 色。
悬停态：整行 `background: #f8fafc`。
禁用态（结果页在未运行前）：opacity 0.35，cursor not-allowed。

分隔线：导航区第 5 项（干预配置）之后加 1px `#f1f5f9` 分隔线，将"配置类"与"运行类"视觉分组。

**进度区**（侧边栏底部，分隔线上方）：

```
配置完成度
████████░░  75%
5/6 项已完成 · 1 项警告
```

进度条高 4px，teal 填充，slate-200 底色。

**Hero 区域细节**

主标题：`意见动力学仿真平台` — 22px/500，line-height 1.3，两行排布（中英文各一行）。
副标题：2 行说明文字，13px，secondary 色，max-width 480px，line-height 1.65。

标签组（Chip 样式）：5 个，横向排列，可换行：
- `有界信任动力学` `Hawkes 级联` `影响力场` `多层意见` `空间分布`
- 样式：11px，`background: #f1f5f9`，`border: 0.5px solid #e2e8f0`，border-radius 12px，padding 3px 10px。

**功能卡片**（3 列，gap 12px）

每卡：`background: #f8fafc`，border `#e2e8f0`，border-radius 6px，padding 14px 16px。
卡内：13px/500 标题 + 12px/400 说明（2 行以内）。
三张卡依次：模型配置 / 实时监控 / 分析报告。

**操作按钮组**

`开始配置实验 →`：lg 按钮，teal 底，白字，width 180px。
`加载预设方案 ▾`：lg 按钮，白底，slate 边框，打开下拉菜单选择预设名。

**预设下拉菜单**：宽同按钮，白底，shadow-sm，每项 32px 高，悬停 `#f8fafc`。选中后按钮文字变为预设名，出现"清除"图标。

**语言切换**

右上角，两个胶囊按钮紧排：选中的为 `background: #0f172a; color: white`，未选中为白底 slate 边框。
切换时所有文字标签同步更新（不重建组件树）。

#### P3 模型配置

**布局**

页面分两栏：左侧导航索引（120px，仅在高度足够时展示）+ 右侧 Accordion 区（flex:1）。

```
┌────────┬─────────────────────────────────┐
│ 章节   │  当前展开的 Accordion            │
│ 索引   │                                 │
│ 9 项   │  (可选：双栏布局，每栏放半屏宽参数) │
│ 竖向   │                                 │
└────────┴─────────────────────────────────┘
```

**左侧章节索引**（可选显示，窄屏隐藏）

每项 28px 高，12px，点击平滑滚动到对应 Accordion，当前所在章节高亮。

**Accordion 改造**

原有 Accordion header 的视觉问题：背景与页面底色相同，层次不够清晰。

改造方案：
- Header 高度从原来的不固定改为 **36px 固定**。
- Header 背景：`#f8fafc`，底部边框 `#e2e8f0`，无圆角（flush 布局）。
- 展开状态：左侧出现 2px teal 竖线，header 文字 primary 色；收起状态文字 secondary 色。
- 内容区 padding：`12px 16px`，白底。

**参数行双栏布局**：同一 Accordion 内，将相关的一对参数（如 epsilon_base 和 mu_base）用 `gr.Row()` 并排，每个占 50%，gap 12px。这与原代码一致，此处确认保留。

**底部操作行**（sticky，吸附在内容区底部）

```
┌─────────────────────────────────────────┐
│  [← 返回]   [保存并继续 →]               │  高度 56px，白底，上边框
└─────────────────────────────────────────┘
```

##### P6-A 参数确认阶段

**布局（两栏）**

```
┌──────────────────────────┬─────────────────┐
│  检查项列表（flex:1.4）    │  参数快照（flex:1)│
│                           │                  │
│  6 条 CheckItem           │  只读 key-value  │
│                           │  表格，等宽字体   │
│  ─────────────────────── │                  │
│  警告汇总 + 操作按钮       │  [复制JSON]      │
└──────────────────────────┴─────────────────┘
```

**CheckItem 行设计**（高度自适应，最小 52px）

```
┌────────────────────────────────────────────────┐
│  [状态图标18px]  分组标题 13px/500              │
│                  详情说明 11px secondary         │
│                                      [去修改]   │
└────────────────────────────────────────────────┘
```

三种状态样式：

`ok`：`background: rgba(22,163,74,0.04)`，`border: 0.5px solid rgba(22,163,74,0.25)`。图标：绿底白钩，18px 圆形。

`warn`：`background: #fffbeb`，`border: 0.5px solid #fde68a`。图标：amber 底白叹号。右侧出现"去修改"文字链接（13px teal，下划线）。

`error`：`background: #fef2f2`，`border: 0.5px solid #fecaca`。图标：red 底白叉。右侧出现"必须修改"链接（13px red）。

**底部操作区**

```
┌──────────────────────────────────────────┐
│  [状态说明：X项通过 · Y项警告 · Z项错误]  │  12px secondary

│  [▶ 确认并启动仿真]   [导出配置 YAML]     │

**参数快照**（右栏）

标题：`参数快照` 12px/500 secondary，右侧 `[复制]` 按钮（sm，点击后变为 `✓ 已复制`，2秒后复原）。

**状态行改造**

原 `status_md` 改为更结构化的布局。左侧：状态胶囊（运行中 teal，完成 green，停止 slate）+ 三个文字指标（Step / Time / Events），用 `·` 分隔，monospace 字体。右侧：暂停和停止按钮紧贴右边，尺寸 sm。

#### P7 实验结果

**布局**

```
┌────────────────────────────────────────────────┐
│  页面标题  [✓ 已完成标签]   操作栏（下载/报告）  │  header
├────────────────────────────────────────────────┤
│  指标卡行（5 个，同 P6-B，只读）                  │  metrics
├────────────────────────────────────────────────┤
│  [动态分析] [静态仪表盘] [特征摘要] [AI 报告]     │  子标签导航
├────────────────────────────────────────────────┤
│                                                 │
│            当前激活子标签内容                     │
│                                                 │
└────────────────────────────────────────────────┘
```

**子标签导航**样式：水平 pill 组（不同于页面间导航的竖向侧边栏）。激活项：`background: #0f172a; color: white`，border-radius 20px，padding 5px 14px。非激活：白底，`border: 0.5px solid #e2e8f0`。

**操作栏**（页面 header 右侧）

三个 sm 按钮紧排：
- `⬇ 下载图表包`（DownloadButton）
- `⬇ 导出特征 JSON`（DownloadButton）
- `📄 生成报告`（Button，触发 AI 报告，切换到 AI 报告子标签并显示进度）

---

##### 子标签 1：动态分析

完整展示最后一次运行的 4 张图表，2×2 布局，尺寸比 P6-B 略大（可利用整个内容区高度）。图表右上角各有一个 `⬇` 小图标按钮，点击下载单张图片。

---

##### 子标签 2：静态仪表盘

单张全宽高质量合成图（`render_dashboard()` 输出），`border-radius: 6px`，`border: 0.5px solid #e2e8f0`，图片撑满容器宽度。图下方：`运行 分析` 按钮（触发完整分析管道），首次进入此标签时自动触发。

---

##### 子标签 3：特征摘要

全宽表格，两列（metric / value），font-family monospace，font-size 12px，行高 28px。表头固定（sticky），背景 `#f8fafc`。偶数行 `#f8fafc` 交替底色。表格上方显示"共 N 项特征"的 muted 文字。

---

##### 子标签 4：AI 报告

若未启用 AI：显示灰色空状态区域，文字"未启用 AI 解析 — 前往分析配置页开启"，含跳转链接。

若已启用但尚未生成：显示 `📄 生成 AI 报告` 大按钮（居中，lg 尺寸）。

生成中：按钮变为 spinner + `生成中…`，禁用状态。

### 四、交互行为规范

#### 页面切换

从任意页面切换到另一页时：
- 当前页 `opacity` 从 1 → 0，duration 80ms
- 新页 `opacity` 从 0 → 1，duration 120ms
- 过渡期间侧边栏不禁用，可连续点击

#### 侧边栏状态点更新时机

触发条件：任意参数输入框失焦（`blur` 事件）+ 页面切换时。不绑定每次 `change` 事件（避免每次击键都触发校验）。

#### 确认启动后

1. 页面切换到 P6-B（监控阶段）
2. 顶栏状态更新为"⚙ 运行中"
3. 侧边栏"实验结果"导航项解除禁用
4. 仿真完成后顶栏更新为"✓ 已完成"，同时侧边栏"实验结果"状态点变绿

#### 表单验证反馈

输入框值超出范围时：`border-color: #dc2626`，输入框下方出现 11px red 错误说明。恢复正常范围后自动清除。此为字段级即时反馈，与 P6-A 的分组级 CheckItem 互补。

#### 预设加载行为

首页或配置页的"加载预设"按钮点击后：
1. 显示确认 dialog（`gr.Modal` 或内联 confirmation row）："加载预设将覆盖当前所有配置，确认继续？"
2. 确认后批量更新所有参数组件值
3. 自动跳转到模型配置页，Accordion 全部展开
4. 侧边栏状态点重新计算

### 五、响应式行为（窄屏适配）

**宽度 < 900px**（平板横屏）

- 侧边栏折叠为 48px 宽，只显示图标，悬停展开 tooltip 显示导航名
- P3 模型配置的左侧章节索引隐藏
- P6-A 两栏布局改为单栏，参数快照移至 CheckItem 列表下方

**宽度 < 640px**（手机 / 竖屏平板）

- 侧边栏完全隐藏，顶栏出现汉堡菜单按钮，点击后侧边栏从左滑入（overlay 模式）
- 所有双列参数行（`gr.Row` 内两个 Number）改为单列堆叠
- 图表 2×2 改为 1×4 纵向堆叠
